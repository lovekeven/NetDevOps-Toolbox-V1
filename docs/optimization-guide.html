01 用户自定义命令功能

高优先级 #1

一、问题背景

原来的系统：用户只能点击按钮执行预设的健康检查，想查其他信息（路由表、ARP表、VLAN信息）没办法！
用户的真实需求：想查设备的路由表、看ARP表有多少条目、看VLAN配置、ping某个地址...

传统做法 vs 现在

步骤 传统做法 优化后
1 打开终端 网页上选设备
2 SSH登录设备 选命令或自己输入
3 输入命令 点执行
4 看结果 看结果
5 退出登录 -

二、核心原理：命令白名单

⚠️ 安全问题：如果让用户随便输入命令，可能会执行危险命令（delete、format、configure），导致设备损坏！

解决方案：命令白名单 - 只允许查询类命令（display、show），禁止配置类命令（configure、delete、format）

# 命令白名单数据结构 COMMAND_WHITELIST = { "h3c": { "display": [ "display interface brief", # ✅ 安全 "display ip routing-table", # ✅ 安全 "display version", # ✅ 安全 ], "ping": ["ping"], # ✅ 安全 }, "cisco": { "show": [ "show ip interface brief", "show ip route", ], } }

硬件 - ✅ 温度、电源、风扇
二层 - ✅ STP、链路聚合

二、核心原理：多厂商命令映射

# 命令映射字典 VENDOR_COMMANDS = { "h3c": { "routing": "display ip routing-table", "arp": "display arp", "vlan": "display vlan", "ospf": "display ospf peer", }, "cisco": { "routing": "show ip route", "arp": "show arp", "vlan": "show vlan", "ospf": "show ip ospf neighbor", } } def get_vendor_command(vendor, command_type): """根据厂商类型获取对应命令""" return VENDOR_COMMANDS.get(vendor, {}).get(command_type)

07 实时监控 WebSocket

中优先级 #7

一、问题背景

传统方式（轮询）：客户端反复请求"有新数据吗？"，浪费资源。
WebSocket方式（推送）：服务器主动推送"新数据来了！"，实时高效。

轮询 vs WebSocket 对比

对比项 轮询 WebSocket
实时性 秒级延迟 毫秒级延迟
资源消耗 高（反复请求） 低（长连接）
服务器压力 大 小
用户体验 一般 流畅

二、代码实现

后端（Flask-SocketIO）

from flask_socketio import SocketIO, emit socketio = SocketIO(app) @socketio.on('connect') def handle_connect(): """客户端连接时""" emit('response', {'data': 'Connected'}) def push_monitoring_data(data): """服务器主动推送""" emit('monitoring_update', data, broadcast=True)

前端（Socket.IO Client）

const socket = io(); socket.on('monitoring_update', function(data) { // 更新图表 updateChart(data); });
