**技术亮点1：WebSocket 实时监控**

**（点击"监控告警" → "实时监控"）**

我们使用 WebSocket 技术实现设备状态的实时推送，而不是传统的轮询方式。

**（展示后端代码 - 文件：web/web_dashboard.py）**

```python
# WebSocket 实时推送监控数据
@socketio.on('start_monitoring')
def handle_start_monitoring(data):
    interval = data.get('interval', 60)
    while is_monitoring:
        # 检查所有设备状态
        devices = check_all_devices_status()
        # 实时推送给前端
        socketio.emit('monitoring_update', {
            'devices': devices,
            'timestamp': datetime.now().isoformat()
        })
        socketio.sleep(interval)
```

**（展示前端代码 - 文件：web/templates/index.html）**

```javascript
// 接收 WebSocket 推送
socket.on('monitoring_update', function(data) {
    // 实时更新图表
    updateChart(data);
    // 实时更新统计
    updateStats(data);
});
```

**优势：** 实时性强、延迟低、节省带宽。

```python
class SDNCollector:
    """SDN 控制器拓扑采集器"""
    def get_topology(self):
        """从 Ryu 控制器获取拓扑"""
        # 获取交换机列表
        switches = self._get_switches()
        # 获取链路信息
        links = self._get_links()
        # 获取主机信息
        hosts = self._get_hosts()
        return self._build_topology(switches, links, hosts)
```

## 【可能的问题】

**Q1：你的AI分析是怎么实现的？**
A：我们使用DeepSeek大语言模型，通过API调用，分析设备健康数据，给出诊断建议。

**Q2：支持哪些设备？**
A：支持H3C、华为、Cisco等主流厂商设备，通过Netmiko库实现多厂商兼容。

**Q3：拓扑发现是怎么实现的？**
A：通过SNMP协议采集设备信息，使用LLDP协议发现邻居关系，支持多层递归扫描。

**Q4：告警是怎么工作的？**
A：告警引擎每60秒检查一次设备指标，超过阈值触发告警，支持邮件通知。

**Q5：项目有多大？**
A：35,000+行代码，71个文件，6大功能模块。
