## 1. 项目概述

### 1.1 项目背景

随着企业网络规模的不断扩大，传统的人工运维方式已经无法满足现代网络管理的需求。网络设备数量激增、拓扑结构日益复杂、故障排查效率低下等问题，迫切需要一套智能化、自动化的网络运维工具。

### 1.2 项目目标

SmartNet 智网工坊旨在为企业提供一站式智能网络运维解决方案，实现：

- **网络拓扑自动发现与可视化**：通过 SNMP/LLDP 协议自动发现网络设备，生成可视化拓扑图
- **设备集中管理**：统一管理多厂商设备，支持批量操作
- **智能监控告警**：实时监控设备状态，异常自动告警
- **AI 辅助分析**：利用人工智能技术辅助故障诊断和运维决策

### 1.3 目标用户

- 网络运维工程师
- 系统管理员
- IT 运维团队

### 3.1 网络拓扑可视化

#### 3.1.1 功能概述

网络拓扑可视化是本项目的核心功能之一，通过 SNMP/LLDP 协议自动发现网络设备及其连接关系，生成可视化的网络拓扑图。

#### 3.1.2 技术实现

**拓扑发现算法**

```
┌─────────────────────────────────────────────────────────┐
│                   拓扑发现流程                            │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  1. 种子设备输入                                         │
│     ↓                                                   │
│  2. SNMP 采集设备信息                                    │
│     ├── 系统信息 (sysDescr, sysName)                    │
│     ├── 接口信息 (ifTable)                              │
│     ├── LLDP 邻居表 (lldpRemTable)                     │
│     └── MAC 地址表 (dot1dTpFdbTable)                   │
│     ↓                                                   │
│  3. 邻居关系解析                                         │
│     ├── LLDP 直接邻居                                   │
│     └── MAC 地址表推导（创新算法）                       │
│     ↓                                                   │
│  4. 多层递归扫描（BFS）                                  │
│     ↓                                                   │
│  5. 拓扑数据生成                                         │
│     ↓                                                   │
│  6. vis.js 可视化渲染                                    │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

**三种扫描模式**

| 模式 | 说明 | 适用场景 |
|------|------|---------|
| 单层扫描 | 只扫描种子设备的直接邻居 | 快速查看局部拓扑 |
| 多层扫描 | 递归扫描全网设备 | 完整拓扑发现 |
| MAC回退 | LLDP失效时用MAC表推导 | HCL模拟器等特殊环境 |

**MAC 双向匹配算法（创新点）**

```python
def build_links_from_mac_table(mac_tables):
    """
    MAC 双向匹配算法
    通过比对不同设备的 MAC 地址表，推导设备连接关系
    """
    links = []
    devices = list(mac_tables.keys())
    
    for i in range(len(devices)):
        for j in range(i + 1, len(devices)):
            dev1, dev2 = devices[i], devices[j]
            
            # 获取两个设备的 MAC 表
            macs1 = mac_tables[dev1]
            macs2 = mac_tables[dev2]
            
            # 双向匹配
            for mac1, iface1 in macs1.items():
                for mac2, iface2 in macs2.items():
                    # 如果一个设备的 MAC 出现在另一个设备的同一接口
                    # 则认为这两个设备相连
                    if mac1 == mac2:
                        links.append({
                            'source': dev1,
                            'source_port': iface1,
                            'target': dev2,
                            'target_port': iface2
                        })
    
    return links
```

- **自动布局**：智能布局算法，自动整理拓扑图
- **导出功能**：支持 PNG、JSON、CSV 多种格式导出

#### 3.2.3 一键设备查询

**功能说明**

一键查询功能可以同时向多台设备发送命令，批量获取设备信息，大大提高运维效率。

**技术实现**

```python
def query_all_devices(commands, email_recipients=None):
    """
    一键查询所有设备
    """
    results = []
    
    # 使用 Nornir 并发执行
    nr = InitNornir(config_file="config/nornir_config.yaml")
    
    for device in nr.inventory.hosts.values():
        try:
            # 建立 SSH 连接
            with ConnectHandler(**device) as conn:
                device_result = {
                    'device_name': device.name,
                    'device_ip': device.hostname,
                    'commands': {}
                }
                
                # 执行所有命令
                for cmd in commands:
                    output = conn.send_command_timing(cmd)
                    device_result['commands'][cmd] = output
                
                results.append(device_result)
        except Exception as e:
            results.append({
                'device_name': device.name,
                'device_ip': device.hostname,
                'status': 'failed',
                'error': str(e)
            })
    
    # 可选：发送邮件报告
    if email_recipients:
        send_query_report(results, email_recipients)
    
    return results
```

from nornir import InitNornir
from nornir_netmiko.tasks import netmiko_send_command

def batch_health_check(device_names):
    """
    批量健康检查
    """
    nr = InitNornir(config_file="config/nornir_config.yaml")
    
    # 筛选指定设备
    filtered_nr = nr.filter(name__in=device_names)
    
    # 并发执行检查
    results = filtered_nr.run(
        task=netmiko_send_command,
        command_string="display health"
    )
    
    return results

## 5. 技术创新点

### 5.1 MAC 双向匹配拓扑推导算法

**创新背景**

在 HCL 等模拟器环境中，LLDP MIB 无法获取，传统拓扑发现方法失效。

**创新方案**

通过比对不同设备的 MAC 地址表，利用二层网络的 MAC 学习机制，推导设备连接关系。

**算法优势**

- 不依赖 LLDP 协议
- 适用于模拟器环境
- 可发现隐藏的连接关系

### 5.2 多扫描模式自适应

提供单层扫描、多层扫描、MAC回退三种模式，用户可根据实际环境选择最优方案。

### 5.3 AI 辅助运维决策

将 AI 技术应用于网络运维，实现：
- 智能故障诊断
- 拓扑智能推测
- 运维报告自动生成

### 5.4 Web SSH 终端集成

将 SSH 终端集成到 Web 界面，实现：
- 无需安装客户端
- 多设备并行管理
- 命令模板快速执行
