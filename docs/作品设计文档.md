## 一、目标问题与意义价值

### 1.1 应用领域

本作品面向企业网络运维领域，为网络运维工程师、系统管理员、IT运维团队提供一站式智能网络运维解决方案。

### 1.2 解决的问题

**核心问题：**

1. **拓扑发现效率低**：传统人工绘制网络拓扑耗时耗力，且容易出错
2. **设备管理分散**：多厂商设备需要不同工具管理，运维复杂度高
3. **故障排查慢**：缺乏智能化诊断手段，故障定位依赖人工经验
4. **运维成本高**：需要安装多种客户端软件，学习成本高

**具体痛点：**

- 网络设备数量多，手动管理效率低下
- 多厂商设备（H3C、Huawei、Cisco）命令不统一
- 传统SSH客户端需要安装，多设备并行管理困难
- 缺乏智能化的故障诊断和运维决策支持

### 1.3 实现目标

1. **自动化拓扑发现**：通过SNMP/LLDP协议自动发现网络设备，生成可视化拓扑图
2. **统一设备管理**：支持多厂商设备统一管理，提供Web SSH终端
3. **智能监控告警**：实时监控设备状态，异常自动告警
4. **AI辅助运维**：利用人工智能技术辅助故障诊断和运维决策

### 1.4 基本功能

| 功能模块 | 功能说明 |
|---------|---------|
| 拓扑管理 | 自动发现网络设备，生成可视化拓扑图 |
| 运维工具 | Web SSH终端、一键设备查询、常用命令集 |
| 监控告警 | 实时监控设备状态，灵活告警规则配置 |
| AI分析 | 设备健康分析、拓扑推测、报告生成 |
| 设备管理 | 设备增删改查、健康检查、配置备份 |

### 1.5 理论意义与应用价值

**理论意义：**

- 探索AI技术在网络运维领域的应用模式
- 研究多厂商设备统一管理的技术方案
- 实现Web终端与网络设备的实时交互

**应用价值：**

- 提高网络运维效率，降低人工成本
- 降低运维门槛，新手也能快速上手
- 适应企业多厂商设备环境，降低管理复杂度
- 为智能化运维提供技术参考

### 2.3 详细设计方案

#### 2.3.1 拓扑发现方案

**技术方案：**

1. **SNMP协议采集**：通过pysnmp库实现SNMP协议，获取设备信息
2. **LLDP邻居发现**：读取设备LLDP邻居表，获取连接关系
3. **多层递归扫描**：从种子设备开始，递归扫描全网设备
4. **vis.js可视化**：使用vis.js库渲染拓扑图

**扫描流程：**

```
种子设备输入 → SNMP采集设备信息 → 获取LLDP邻居 → 递归扫描邻居 → 生成拓扑数据 → 可视化渲染
```

#### 2.3.2 运维工具方案

**Web SSH终端：**

- 前端：xterm.js终端模拟器
- 后端：paramiko SSH连接库
- 通信：WebSocket实时数据传输

**一键设备查询：**

- 并发框架：Nornir
- 执行方式：多线程并发执行
- 结果处理：统一格式化展示

#### 2.3.3 监控告警方案

**监控方式：**

- 定时任务：APScheduler定时执行
- 数据采集：SNMP/SSH获取设备状态
- 实时推送：WebSocket推送监控数据

**告警机制：**

- 规则配置：支持CPU、内存、接口等指标
- 阈值设置：支持大于、小于、等于等运算符
- 通知方式：邮件通知

#### 2.3.4 AI分析方案

**技术方案：**

- AI模型：DeepSeek大语言模型
- 调用方式：API调用
- 应用场景：健康分析、拓扑推测、报告生成

### 3.2 功能实现

#### 3.2.1 拓扑管理功能

**核心代码：**

```python
def build_topology(seed_ip, community='public', max_depth=3):
    """
    构建网络拓扑
    """
    visited = set()
    queue = [(seed_ip, 0)]
    nodes = []
    links = []
    
    while queue:
        ip, depth = queue.pop(0)
        
        if ip in visited or depth > max_depth:
            continue
        
        visited.add(ip)
        
        # SNMP采集设备信息
        collector = SNMPCollector(ip, community)
        device_info = collector.get_system_info()
        neighbors = collector.get_lldp_neighbors()
        
        nodes.append({
            'id': ip,
            'name': device_info.get('sysName', ip),
            'type': detect_device_type(device_info),
            'ip_address': ip,
            'status': 'online'
        })
        
        # 处理邻居关系
        for neighbor in neighbors:
            remote_ip = neighbor.get('remote_ip')
            if remote_ip and remote_ip not in visited:
                queue.append((remote_ip, depth + 1))
                links.append({
                    'source_node': ip,
                    'source_port': neighbor.get('local_port'),
                    'target_node': remote_ip,
                    'target_port': neighbor.get('remote_port'),
                    'status': 'up'
                })
    
    return {'nodes': nodes, 'links': links}
```
